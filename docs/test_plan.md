# TradingView 模拟交易功能测试计划

## 1. 概述
- **项目背景**：本计划旨在对 TradingView 的模拟交易核心功能进行系统化测试，以实践金融业务测试全流程。
- **测试目标**：验证“行情查看”、“模拟交易下单”、“持仓管理”三大核心业务流程的正确性、稳定性和用户体验。

### 2. 测试范围
#### 包含的功能模块：
1.  **行情图表模块**
    *   功能点：交易对搜索（如输入`BTCUSD`）、切换K线周期（1分钟/1小时等）、叠加技术指标（如MA均线）。
    *   测试焦点：数据准确性、图表响应速度、切换是否流畅。

2.  **模拟交易模块（核心）**
    *   **订单功能**：
        *   市价单：立即成交的正确性。
        *   限价单：挂单、成交、撤销流程。
        *   订单修改：修改挂单的价格和数量。
    *   **订单类型**：区分`BUY`（买入）与`SELL`（卖出）。
    *   **测试焦点**：订单逻辑正确性、资产计算准确性、异常输入处理。

3.  **账户管理模块**
    *   功能点：虚拟资金显示、持仓列表、浮动盈亏计算、订单历史记录。
    *   测试焦点：数据一致性（如现金+持仓=总资产）、记录完整性。
    


### 3. 测试策略与用例设计
| 测试类型 | 针对功能 | 设计思路与方法 | 预期产出 |

## 7. 风险与应对
- **风险1**：模拟平台功能不稳定或更新。
    - *应对*：每日开始前进行冒烟测试，确认核心功能可用。
- **风险2**：网络延迟导致交易指令执行反馈慢。
    - *应对*：记录并分析延迟数据，作为性能测试观察点。
