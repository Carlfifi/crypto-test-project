# TradingView 模拟交易功能测试计划

## 1. 概述
- **项目背景**：本计划旨在对 TradingView 的 Paper Trading（模拟交易）功能进行系统化测试，以实践金融业务测试全流程。
- **测试目标**：验证“行情查看”、“模拟交易下单”、“持仓管理”三大核心业务流程的正确性、稳定性和用户体验。
- **测试特性**：聚焦数字货币交易场景，强调数据准确性、订单状态流转和风险控制。

## 2. 测试范围
- **包含的功能模块**：
    1.  **行情模块**：交易对搜索与切换、价格/K线图展示。
    2.  **交易模块**：
        - 市价单：买入、卖出。
        - 限价单：挂单、撤销。
    3.  **账户模块**：虚拟资金显示、持仓列表、订单历史、浮动盈亏计算。
- **不包含的功能**：真实支付、社交功能、多账户管理。

## 3. 测试策略
| 测试类型 | 测试重点 | 采用方法与工具 |
| :--- | :--- | :--- |
| **功能测试** | 核心业务流程正确性 | 等价类划分、边界值分析、场景法 |
| **异常测试** | 系统对非法输入的防护 | 输入负数、零、超大数量、无效价格等 |
| **业务逻辑测试** | 金融规则符合性（如禁止卖空） | 设计超出持仓的卖出订单 |
| **数据一致性测试** | 资金、持仓、订单金额计算准确 | 逐笔核对订单成交记录与账户变动 |
| **状态流转测试** | 订单生命周期（挂单、成交、撤销） | 跟踪并验证订单状态变化 |

## 4. 测试资源与环境
- **测试环境**：TradingView 公开图表页面 (`https://www.tradingview.com/chart`) 中的 Paper Trading 功能。
- **浏览器**：Chrome 最新稳定版。
- **测试工具**：Postman (API校验)， Python 3.11+ (辅助脚本)， Git (版本管理)。
- **测试数据**：平台提供的初始虚拟资金（$100,000）。

## 5. 交付物
1.  本测试计划
2.  详细的测试用例表 (`test_cases.xlsx`)
3.  探索性测试笔记 (`exploratory_notes.md`)
4.  测试总结报告 (`test_summary_report.md`)

## 6. 风险与应对
- **风险1**：模拟平台功能更新或暂时不可用。
    - *应对*：每日开始前进行冒烟测试；准备备选测试平台。
- **风险2**：网络延迟可能导致订单反馈慢，影响测试效率。
    - *应对*：记录延迟情况，将其作为非功能性观察点纳入报告。
- **风险3**：对复杂金融业务规则理解有误。
    - *应对*：对不确定的测试点（如限价单成交逻辑），先小范围实验，确认后再设计正式用例。
